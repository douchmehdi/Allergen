Rapport ProjetOuaibe - Mineure AppliOuaibe

Mehdi Douch - 31/03/2017

1. Description du projet Allergen

Beaucoup de personnes souffrent de problèmes d’intolérances alimentaires et de pathologies telles que le SCI/colopathie fonctionnelle. Chez ces personnes la consommation de certains aliments provoque l’apparition de symptômes qui minent la qualité de vie. Certains aliments sont connus pour être particulièrement mal absorbés par les personnes souffrant de SCI : les FODMAP. Cependant toutes les personnes ne sont pas égales devant les FODMAPs et il est fréquent que certaines personnes souffrant de SCI tolèrent parfaitement une partie de ces aliments. Le régime sans FODMAP étant particulèrement restrictif il est dans l’intérêt de chaque malade de determiner leplus précisement possible le champ des aliments qu’il tolère et qu’il ne tolère pas afin de pouvoir vivre la vie la plus normale possible. Déterminer ce périmètre passe souvent par le suivi d’un régime dit d’ « élimination » pendant lequel les personnes élimine une partie des FODMAP pendant une période de quelques semaines afin de déterminer si l’élimination conduit à une amélioration de l’état de santé général.

Le projet Allergène est un projet qui vise à aider les personnes souffrantes de SCI en leur permettant de prendre note de ce qu’ils mangent et des symptômes qu’ils présentent afin de les aider à déterminer le périmètre de ce qu’ils peuvent manger tout en restant en bonne santé et améliorer leur état de santé général au quotidien.

Le projet allergène est un projet que j’ai envie de réaliser depuis un certain temps car je souffre, tout comme ma maman de’intolérances alimentaires et je n’ai trouvé aucun outil qui facilite le régime d’élimination.

2. Objectif de l’application Allergen

  • Page 1: Enregistrement des repas et de la composition de celui-ci
  • Page 2 : Enregistrement des symptômes
  • Page 3 : Pour une période de temps donnée, être capable de restituer les allergènes sur un graphique dont l’abcisse serait le temps et représenterait de manière graphique les allergènes absorbés par l’utilisateur ainsi que les symptomes digestif pendant ce même labs de temps.

Dans une version plus avancée il y aura la possibilité de démarer un régime d’élimination et d’être guidé par l’application

3. Architecture du projet

Le projet sera composé d’une API qui sert les données d’une part et d’une application web qui affiche les données et est l’interface avec l’utilisateur d’autre part.

Afin de réaliser le projet, le framework Node.js Express a été utilisé.

5. Fonctionnement de l’API :

L’API devra être en mesure de fournir les services suivants :

  • créer un repas à partir d’une liste d’aliments et d’une date fournie par l’utilisateur
  • restituer les repas consommées par l’utilisateur sous forme d’un tableau
  • de vérifier si le username et le password fournis par l’utilisateur sont bien dans la base de données et de créer un JsonWebtoken qui sera stocké sur le localStorage de l’utilisateur. Le token permettra à l’utilisateur d’accéder à ses données de manière sécurisée.

Ces fonctionnalités peuvent être trouvées dans la partie « routes/api.js »

6. CSS

Deux templates ont été utilisés :

  • le template ‘Shield’ pour la page d’acceuil qui a pour but d’informer.
  • le template ‘Dashgum-free’ pour le coté application orienté utilisateur.
